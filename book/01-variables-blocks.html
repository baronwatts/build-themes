Tumblr themes have two operators, which allow you to display user data in your themes, called variables and blocks.

These operators are placed in your theme’s HTML, which is then parsed by Tumblr’s servers.

Variables

Variables refers to data which should be outputted. Variable names are wrapped in curly brackets (also known as braces), for example {variable name}.

One example of a variable, defined in the theme documentation, is the Title variable. To use the blog’s title in HTML you would type:

<a href="/">{Title}</a>

Since titles of blogs vary, you should not hard code a specific title, instead use a variable.

On this website (which I have named ‘Build Themes’ in the customize panel), the theme code is processed server-side by Tumblr, and then outputted in the client-side HTML as:

<a href="/">Build Themes</a>

Notice how the user never sees the variable you used to display the title, variables are transformed into the actual data by a server.

Blocks

Blocks are used to iterate through a set of data and/or conditionally render HTML (that is, if something is or isn’t available).

Iterating through data

The first use of blocks is to loop through a set of data.

Here’s an example that uses {block:Posts} to display “Posted on Tumblr” on every single post:

{block:Posts} <p>Posted on Tumblr</p> {/block:Posts}

This use of blocks is telling the Tumblr servers that for each post in the posts data, some text should be displayed (namely, “Posted on Tumblr”).

Instead of manually adding the desired HTML markup to every single post, blocks are used to reduce duplicate code.

Conditionally rendering

Blocks can also render on a simple condition which tests if data are available.

For example, you may use a block to see if the blog has any text posts to display.

Building on the previous example, we can limit to rendering the text if the post is a photo post.

{block:Posts}
{block:Photo}
<p>Posted on Tumblr</p>
{/block:Photo}
{/block:Posts}

This comes very useful when we want to render different content for each post type.

Putting it all together

This is the HTML used for rendering pagination on Tumblr blogs.

Pagination refers to the ‘next page’ and ‘previous page’ links commonly seen on blogs, and allows for navigation between older and newer posts.

{block:Pagination}
<nav class="pagination">
  <ul>
    {block:PreviousPage}<li><a href="{PreviousPage}">{lang:Previous}</a></li>{/block:PreviousPage}
    {block:NextPage}<li><a href="{NextPage}">{lang:Next}</a></li>{/block:NextPage}
  </ul>
</nav>
{/block:Pagination}

Firstly, {block:Pagination} asks “do we have more than one page?”. If this is true, the following markup is processed.

{block:NextPage} asks “do we have a next page?”. This only renders if there is more content to load. For example, if the reader is on the last page of content, there is no need to display a “next page” link.

{block:PreviousPage} asks “do we have a previous page?”. This only renders if there is a page to go back to. This is helpful if you are already on page one of a blog, since there is no “page zero” to visit.

{PreviousPage} and {NextPage} are variables for the hyperlink, relative to what page the reader is on. For example, if the reader is on the third page of the blog, {PreviousPage} links to the second page, and {NextPage} links to the fourth page.

Documentation

Tumblr keeps a list of variables and blocks which can be used in the theme documentation.

What you should know

  • Variables are used to output user data
  • Blocks are used to iterate through data or test a condition
